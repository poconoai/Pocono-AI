For IT Directors & Technical Auditors

Network Topology: Inbound vs. Outbound — The Full Picture

Zero exfiltration does not mean zero network connectivity. It means zero outbound data transmission during inference and operation. A skeptical IT director deserves a precise answer to the obvious question: if it's isolated, how do model weights and security patches get in?

⊘ Permanently Blocked — Outbound
  • All outbound internet routes blocked at VLAN boundary
  • No PHI transmission during inference — ever
  • No telemetry, heartbeat, or phone-home to vendor
  • No cloud API calls for model inference or retrieval
  • No outbound logging to external SIEM
✓ Permitted — Controlled Inbound
  • Authorized clinical workstations on isolated VLAN (inbound only)
  • EHR PDF export via secure local share — staff-initiated
  • Model & patch updates: cryptographically signed USB only
  • LLM weight delivery: signed package, SHA-256 verified before apply
  • Unsigned or unverified packages are automatically rejected
LLM Weight & Patch Update Protocol — How It Actually Works
①
Package Prepared
Pocono AI signs model weights or patch with Ed25519 private key. SHA-256 hash published in the secure client portal.
②
USB Delivery
Signed package loaded onto encrypted USB. IT administrator or practice manager receives and physically inserts into the Sentinel Node.
③
Signature Verified
Sentinel Node verifies Ed25519 signature and SHA-256 hash before any package touches the filesystem. Mismatch = automatic rejection, no install.
④
Applied & Logged
Update applied offline. Every install event is logged to the 90-day immutable local audit trail — timestamp, package hash, and installer identity.

The result: LLM weights, tuning, and security patches enter the system only via cryptographically signed, human-carried USB media. An unsigned USB is physically ignored by the system. No remote management interface. No vendor back-door. No automatic cloud pull. The air gap is not a policy — it is enforced by the update architecture itself.
