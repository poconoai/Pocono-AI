The 3-Tier Clinical RAG Framework

The Sentinel Node is not one tool — it's a role-aware AI deployment that serves every member of your practice staff differently. Each tier accesses the same locally-stored patient data, but the query types, verification requirements, and output formats are calibrated to the clinical and administrative responsibilities of each role.

Tier A · Licensed Providers

Clinical Decision Support

Physicians and NPs query the Sentinel Node for cited patient history summaries before appointments, during care coordination, and for referral documentation. Every response cites the retrieved EHR document chunk — enabling Human-in-the-Loop verification before any clinical decision.

  • Pre-visit chart summaries with source citations
  • Differential support from indexed records
  • Referral package generation
  • All outputs require physician review before use
Tier B · Clinical Support Staff

Prep, Triage & Prior Auth

Medical assistants and nurses use the Sentinel Node to build complete prior authorization submissions from locally-indexed patient records — reducing the 82% wrongful denial rate by ensuring submissions include all supporting documentation on the first attempt.

  • Prior auth package assembly from local records
  • Medication history retrieval and verification
  • Intake triage documentation prep
  • Lab result context for nursing follow-up
Tier C · Administrative Staff

Operations & Billing Support

Front-desk and billing staff use the Sentinel Node to retrieve relevant policy context, code lookups, and patient history for billing inquiries — without accessing the full clinical record. Role-based query scoping keeps the right data in front of the right staff member.

  • Billing inquiry context retrieval
  • Insurance policy document queries
  • SLA and procedure code lookups
  • Patient communication history summaries
One hardware deployment. One local database. Three role-aware access tiers. No PHI leaves your building at any tier. Every query is logged in the 90-day immutable audit trail. Role-based access configuration is scoped during the 60-Day Proof of Value engagement.
